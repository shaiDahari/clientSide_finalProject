## PDF Formatting Requirements

### Critical Requirements:
1. **No Broken Lines:** Ensure code lines are not split across pages
   - Use smaller font size if needed
   - Adjust page margins
   - Use landscape orientation if necessary

2. **File Names:** Each file must have its name clearly displayed
   - Use header or separator before each file
   - Format: `FILE: [path/to/file.jsx]`

3. **Code Formatting:**
   - Maintain original indentation
   - Preserve spacing
   - Keep code readable
   - Use monospace font for code

4. **Organization:**
   - Group files by directory
   - Use clear separators between files
   - Add table of contents if PDF is long

5. **PDF Naming:**
   - Format: `[firstname]_[lastname].pdf`
   - Example: `john_smith.pdf`
   - Use lowercase only
   - Use underscore between names

## Suggested PDF Creation Method

### Option 1: Using Word/Google Docs
1. Create document with header information
2. For each code file:
   - Add file path as heading
   - Paste code in monospace font
   - Ensure no line breaks
   - Add page break if needed
3. Export to PDF
4. Verify no lines are broken

### Option 2: Using LaTeX
1. Use `listings` package for code
2. Set `breaklines=false` to prevent line breaks
3. Use landscape mode if needed
4. Compile to PDF

### Option 3: Using Markdown to PDF
1. Create markdown file with code blocks
2. Use tools like Pandoc or Markdown PDF
3. Configure to prevent line breaks
4. Export to PDF
